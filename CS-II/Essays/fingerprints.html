HTTPS, or Hypertext Transfer Protocol Secure, is used to establish a secure connection on what is widely known as, the Internet. Security and privacy are fundamental to users all across world, in order to protect the user’s data HTTPS connections help provide both. HTTPS also goes by different names such as SSL(Secured Socket Layers) and TLS(Transport Layer Security). HTTPS connection is an extension of it predecessor HTTP. The main difference between HTTPS connection and HTTP connection is that HTTPS is more secure and utilizes TLS encryption allowing it to be safe and secure. These advancements have made it difficult to perform MITM (Man-In-The-Middle Attack) which is when an attacker places himself between the user and application in order to intercept any data. An HTTPS connection makes it difficult for the attacker to “eavesdrop” on the user (Gibson). In order to be considered secure by the HTTPS protocol you must have a digital certificate signed by the CA’s (Certificate Authorities) of that website. However, there are cases where individuals are not pleased with the idea of secure connection. As coined by the FBI it is referred to as the “Going Dark Problem” which states that you see less and less of what is going on or a decrease in visibility of the Internet. A problem stated by many private institutions, corporations, schools, and organizations. Due to this, these organizations also have the ability to eavesdrop on individuals through a technology known as HTTPS Proxy Appliances. This allows institutions to bypass the certification. The risk of this technology can leave you prone to something called SSL Interception which is a proxy application that establishes a second secure connection to the remote web server, however this time they are able to impersonate the user (Gibson). SSL Interception cannot be prevented but it can be detected due to the fact that it is very hard to trick another security certificate. When examining a website you can create a hash, a complex mathematical algorithm that processes what they have acquired. By changing a single bit inside of the certificate, half of the fingerprints hash bits will also change. These fingerprints are useful in detection and are able to locate any changes that occur in the security certificate. Unfortunately, the system is not perfect is prone to errors called false positives or false negatives. A false positive states you’ve identified an attack when there is not one and a false negative is when you detect something as not an attack when there is one (Gibson).
